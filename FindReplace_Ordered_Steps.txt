==============================================================================
AFTER ALL STEPS:
  1. VBA IDE → Debug → Compile (zero errors = success)
  2. If errors remain: check the error line, it will be a pointer/handle mismatch
  3. Common post-fix errors:
     - "Type mismatch": A Long variable receiving a LongPtr function result
       → Change the variable declaration to LongPtr
     - "Expected: expression": Usually a syntax error from bad replace
       → Undo (Ctrl+Z) that specific replace and redo manually
     - Comparison of LongPtr to 0: Use "= 0" or "= CLngPtr(0)" — both work
==============================================================================
QUICK RULE REMINDER:
  LongPtr = Any Windows HANDLE, pointer, hWnd, hKey, pidl, lParam, wParam
  Long    = Counts, flags, boolean returns (BOOL), error codes, string lengths
